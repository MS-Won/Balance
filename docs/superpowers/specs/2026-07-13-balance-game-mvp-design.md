# "오늘의" 밸런스 게임 — MVP 설계 문서

- 작성일: 2026-07-13
- 범위: 핵심 MVP (투표/실시간 채팅/인정/대표의견/명예의 전당). 카카오톡 공유, 광고 실연동, 사용자 문제 제안 플랫폼은 이후 단계로 분리.

## 1. 목적

URL로 접속하는 밸런스 게임 페이지. 매일 하나의 밸런스 게임 주제가 제시되고, 사용자는 두 선택지(A/B) 중 하나의 진영을 골라 투표하고 같은 진영·반대 진영 사용자와 실시간으로 채팅하며 논쟁한다. 투표는 언제든 변경 가능하다. 매일 자정(KST)에 그날의 결과가 확정되고 다음 밸런스 게임이 시작된다.

## 2. 아키텍처

- **프론트엔드**: Next.js, Vercel에 배포. 반응형(모바일 우선) 웹.
- **백엔드/DB/실시간**: Supabase (Postgres + Realtime 구독 + Scheduled Function).
- **사용자 식별**: 별도 회원가입 없음. 브라우저 localStorage에 저장하는 익명 `device_id`(UUID)와 최초 1회 입력받는 닉네임을 사용. 같은 브라우저에서는 다음 방문 시에도 유지되지만, 기기·브라우저를 바꾸면 새 사용자로 취급된다. 어뷰징 방지(쿠키 삭제 후 재투표 등)는 이번 범위에서 다루지 않는다.
- **관리자**: 비공개 `/admin` 경로, 비밀번호 기반의 간단한 세션 보호. 별도 계정 시스템 없음.

## 3. 데이터 모델

| 테이블 | 주요 컬럼 | 설명 |
|---|---|---|
| `balance_games` | id, date, choice_a_label, choice_b_label, description(상세가정), status(`scheduled`/`active`/`ended`) | 하루에 하나씩 활성화되는 밸런스 게임 문제. 관리자 페이지에서 미리 등록. |
| `votes` | game_id, device_id, choice(`A`/`B`), updated_at | `(game_id, device_id)` unique. 투표 변경 시 upsert. |
| `chat_messages` | id, game_id, device_id, nickname, choice(작성 당시 소속 진영), content, created_at | 투표 진영이 바뀌면 이후 신규 메시지의 choice도 즉시 전환된 값을 사용. 과거 메시지의 choice는 작성 당시 값을 유지. |
| `endorsements` | message_id, device_id | `(message_id, device_id)` unique로 중복 인정 방지. 메시지당 사용자 1명 1회. |
| `hall_of_fame` | id, game_id, date, winning_choice, message_id, nickname, endorsement_count | 자정 롤오버 시 그날의 승리 진영 대표의견 작성자를 추가. |

인정 수는 `endorsements`를 message_id로 집계해 실시간 구독으로 반영한다 (카운트 캐시 컬럼은 필요 시 추가 최적화로 남겨둔다).

## 4. 핵심 기능 흐름

### 4.1 투표 & 진영 전환
- 사용자는 A 또는 B를 선택해 투표하고, 그 순간부터 해당 진영 사용자로서 채팅에 참여할 수 있다.
- 투표는 언제든 반대 진영으로 변경 가능하며, 변경 즉시 채팅 작성 진영도 함께 전환된다. 이미 작성한 과거 채팅의 진영 표시는 바뀌지 않는다.
- 득표 그래프는 퍼센티지와 득표수를 함께 표시하며(예: `62% (1,240표)`), Supabase Realtime으로 실시간 갱신된다.

### 4.2 채팅 & 인정
- 채팅은 통합 피드(하나의 스트림)로 표시하되, 각 메시지에 진영 태그(색상/아이콘)를 붙여 구분한다.
- 채팅창 우측의 '인정' 버튼을 누르면 인정 수가 실시간으로 집계된다. 사용자 1명당 메시지 1개에 1회만 인정 가능(취소만 가능, 중복 인정 불가).
- 채팅 리스트는 기본적으로 최신순 실시간 스트림이며, '인정순 보기' 토글로 인정 많은 순 정렬로 전환할 수 있다.

### 4.3 대표의견 (진영별 Top1)
- 각 진영에서 인정 수가 가장 많은 메시지 1개를 실시간으로 계산해, 채팅창 최상단에 좌우 2분할로 고정 표시한다 (`🅰 대표의견` / `🅱 대표의견`).
- 대표의견은 실시간으로 바뀔 수 있다 (다른 메시지가 인정을 더 받으면 교체).

### 4.4 자정(KST) 롤오버
매일 00:00 KST, Supabase Scheduled Function(pg_cron)이 다음을 수행한다:
1. 활성 게임의 최종 득표를 집계해 승리 진영을 결정한다.
2. 승리 진영의 대표의견(그 진영 내 인정 수 1위 메시지) 작성자를 `hall_of_fame`에 추가한다.
3. 활성 게임의 status를 `ended`로 변경하고, 다음으로 예약된 게임을 `active`로 전환한다.
4. 종료된 게임의 채팅/투표 화면은 사라지고, "어제의 결과" 요약(승리 진영, 최종 득표율, 명예의 전당 등극자)만 별도 화면에서 조회 가능하다. 채팅 원문은 DB에 보관되지만 UI에는 다시 노출하지 않는다.

### 4.5 명예의 전당 규칙
- 하루에 최대 1명만 새로 추가된다 (그날 승리 진영의 대표의견 작성자).
- 화면에는 인정 수 기준 상위 10명만 표시되는 올타임 리더보드다. 즉 11일째부터는 그날 신규 등극자의 인정 수가 현재 10위보다 낮으면 명단에 들지 못하고, 높으면 가장 낮은 순위를 밀어낸다.

## 5. 화면 구성

- **상단**: "오늘의 밸런스 게임" 타이틀 + 자정까지 남은 시간 카운트다운.
- **메인**: 득표 그래프(퍼센티지+득표수) → 진영별 대표의견 고정 바 → 통합 채팅 피드(진영 태그 + 인정순 보기 토글) → 채팅 입력창.
- **하단(스택형)**: 명예의 전당(가로 스크롤, 최대 10명) → 광고 배너 영역(자리만 확보, 실제 연동은 Phase 2).
- 레이아웃은 모바일 세로 화면을 기준으로 하고, 데스크톱에서는 중앙 정렬된 단일 컬럼으로 확장한다.

## 6. 관리자 페이지

- `/admin` 비공개 경로, 비밀번호로 보호.
- 밸런스 게임 문제(선택지 A/B 라벨, 상세가정, 게시 예정 날짜) 등록·수정·삭제 기능.

## 7. 이번 범위에서 제외 (Phase 2 이후)

- 카카오톡 공유 기능 (OG 썸네일에 "오늘의 밸런스 게임" 내용 포함).
- 광고 실제 연동 및 수익화.
- 사용자가 밸런스 게임 문제를 제안하는 플랫폼.
- 어뷰징 방지(디바이스 조작을 통한 중복 투표 등).

## 8. 검증 방법

- 로컬에서 Next.js dev 서버 + Supabase 로컬/개발 프로젝트로 투표 변경, 채팅 작성, 인정 클릭, 대표의견 교체가 실시간으로 반영되는지 두 개 이상의 브라우저 탭으로 확인.
- Supabase Scheduled Function을 수동으로 트리거해 자정 롤오버 로직(승리 진영 판정, 명예의 전당 추가, 다음 게임 활성화)이 올바르게 동작하는지 확인.
- 관리자 페이지에서 문제를 등록하면 다음 자정에 해당 문제가 정상적으로 노출되는지 확인.
