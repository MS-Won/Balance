# 오늘의 밸런스 게임 — 성장/마케팅 전략 설계 (2026-07-15)

## 배경 및 목표

- 현재 상태: 무트래픽, 소액 광고 예산(수만 원 단위) 있음, 개인적으로 영향력 있는 채널/팔로워는 없다고 가정(0에서 시작).
- 성공 기준: 매일 실시간 채팅이 돌아가는 수준(수십~수백 DAU). 특정 숫자 자체보다 "매일 채팅이 살아있는 상태"가 목표.
- 핵심 제약: 이 서비스는 실시간 채팅형 제품이라, 트래픽이 **동시에** 몰려야 의미가 있다. 하루 종일 산발적으로 들어오는 방문자는 채팅을 살아있게 만들지 못한다.

이 제약 때문에 전략은 두 축으로 구성된다:

1. **Track A — 커뮤니티 집중 발사**: 트래픽을 특정 시점에 몰아서 터뜨리는 채널 전략.
2. **Track B — 리텐션 훅**: 한 번 유입된 사람을 다음날도 돌아오게 만드는 장치.

Track A만 하면 트래픽이 반짝하고 사라지고, Track B만 하면 애초에 알림을 받을 사람이 없다. 두 축을 함께 가져가는 것이 이번 설계의 핵심이다.

범위 밖(향후 후속 과제로 미룸): 나노인플루언서 협업, 알림톡 자동화(API 연동), 유료 배너 광고, 트래픽이 실제로 쌓인 이후의 확장 전략.

## Track A: 커뮤니티 집중 발사 전략

**타겟 커뮤니티**: 10대 후반~30대, 남녀 구분 없는 넓은 대중 커뮤니티. 예: 에펨코리아 유머게시판, 디시인사이드 관련 갤러리, 네이트판/여성시대, 인스타그램·스레드의 밈/밸런스게임 계정.

**빈도**: 주 1~2회. 매번 그날 게임 중 가장 자극적이고 바이럴성 있는 질문을 골라 발사한다. 매일 올리면 어느 커뮤니티든 자극/어뷰징으로 신고·차단될 위험이 있다.

**로테이션**: 같은 커뮤니티엔 주 1회 이하로만 올려서 피로도와 신고 위험을 관리한다. 여러 커뮤니티를 돌려가며 사용한다.

**광고 예산 집행 원칙 — "선검증 후증폭"**: 미리 광고를 태우지 않는다. 커뮤니티에 올린 포스트 중 유기적으로 반응이 좋았던 것만 그때 소액으로 인스타그램/스레드에서 부스트한다. 매일 조금씩 광고비를 뿌리는 것보다, 반응이 검증된 콘텐츠에 집중하는 쪽이 효율적이다.

**타이밍**: 모바일 트래픽이 몰리는 저녁 8~11시에 발사한다. 그날 admin에 게임을 등록하는 것과 같은 리듬으로 움직인다.

## Track B: 카카오톡 채널 리텐션 훅

목표: 한 번 유입된 사용자가 다음날도 돌아오게 만든다.

- **채널 개설**: 카카오톡 채널을 무료로 개설한다. 사업자등록이나 대행사 계약은 필요 없다 (알림톡과 달리 친구톡은 채널 관리자센터 웹 UI에서 채널의 '친구'들에게 직접 발송 가능).
- **앱 변경(코드, 최소 범위)**: 앱에 카카오 공식 "채널 추가" 버튼/링크를 하나 삽입한다. 위치는 `ShareBar` 근처 또는 명예의 전당 하단.
- **운영(코드 아님, 수동)**: 매일 admin에서 그날의 게임을 등록할 때, 같은 루틴으로 카카오톡 채널 관리자센터에서 친구톡을 수동으로 한 번 발송한다 (예: "오늘의 밸런스 떴어요! [링크]"). API 연동이나 자동 발송 인프라는 이번 범위에 포함하지 않는다 — 지금 admin에 게임을 수동 등록하는 것과 동일한 운영 패턴을 그대로 확장한 것이다.
- **범위 밖**: 알림톡, 자동 발송 API 연동(비즈메시지 대행사 계약 포함). 트래픽이 실제로 쌓이기 시작하면 그때 자동화를 재검토한다.

## Track B-2: 공유 카드 개인화

현재 카카오톡 공유(`src/components/ShareBar.tsx`)는 "오늘의 밸런스 게임에 참여해보세요"라는 일반 문구와 날짜 기반 OG 이미지만 보여준다. 받는 사람 입장에서 눌러볼 개인적인 이유가 약하다. 개인화하면 공유/클릭률이 올라간다.

- **동작**: 사용자가 투표한 뒤 공유하면, 공유 URL에 자신의 선택(A/B)을 쿼리 파라미터로 포함시킨다 (예: `?d=2026-07-15&c=A`). 기존에 날짜별로 동적 생성되는 오픈그래프 이미지 라우트(`/opengraph-image`)를 확장해서, 이 파라미터가 있으면 "나는 A파! 오늘 A파 62% vs B파 38%"와 같은 개인화된 이미지를 생성한다.
- **왜 카카오톡에서 동작하는가**: 카카오는 미리보기를 URL 단위로 스크래핑·캐싱한다. 사용자마다 파라미터가 다른 URL을 공유하므로, 각자의 공유 링크가 각자의 개인화된 카드로 캐싱된다 — 받는 사람이 아니라 **보내는 사람 기준의 개인화**이며, 별도의 수신자별 추적 인프라가 필요 없다.
- **폴백**: 투표하지 않은 상태로 공유하는 경우, 개인화 파라미터 없이 기존 일반 카드로 폴백한다.
- **범위**: 선택과 비율 정도의 텍스트/숫자 개인화만 다룬다. 카드의 비주얼 디자인 변경은 범위 밖이다.

## 측정 루프

포스팅마다 URL에 짧은 source 파라미터(예: `?src=fmkorea`)를 붙여서 어떤 커뮤니티/포스트가 실제 유입과 채팅 참여로 이어지는지 확인한다. 별도 분석 툴 도입 없이, 이 파라미터를 기준으로 한 간단한 카운트로 충분하다. 반응이 좋았던 채널/주제에 다음 주 발사를 집중시키는 근거로 쓴다.

## 다음 단계

이 스펙 중 코드 변경이 필요한 항목(Track B의 채널 추가 버튼, Track B-2의 공유 카드 개인화)에 대해 구현 계획을 작성한다. Track A(커뮤니티 발사 전략)와 Track B의 수동 친구톡 발송은 코드가 아닌 운영 루틴이므로, 구현 계획이 아니라 실행 체크리스트로 별도 관리한다.
